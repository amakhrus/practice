# Deploying to PythonAnywhere

This guide walks through hosting the GED & SAT LMS on **PythonAnywhere**, which is
the simplest host for this project because it keeps the SQLite database on a
persistent filesystem (your data and student accounts survive restarts).

The code is already production-ready: `DEBUG`, `SECRET_KEY`, `ALLOWED_HOSTS`,
`CSRF_TRUSTED_ORIGINS`, and static-file serving (via WhiteNoise) are all driven by
environment variables, with safe defaults for local development.

---

## 1. Create an account

Sign up at **https://www.pythonanywhere.com** (the free "Beginner" tier is enough
to start). Your site will be at `https://YOURNAME.pythonanywhere.com`.

## 2. Get the code onto the server

Open a **Bash console** from the PythonAnywhere dashboard, then either clone from
GitHub (recommended) or upload a zip.

```bash
git clone https://github.com/YOURUSER/ged-sat-lms.git
cd ged-sat-lms
```

## 3. Create a virtual environment and install dependencies

```bash
python3.11 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

## 4. Create the production `.env` file

In the project folder, create a `.env` file (it is git-ignored, so it never leaves
the server):

```bash
cat > .env <<'EOF'
DJANGO_SECRET_KEY=PASTE_A_LONG_RANDOM_VALUE_HERE
DJANGO_DEBUG=False
DJANGO_ALLOWED_HOSTS=YOURNAME.pythonanywhere.com
DJANGO_CSRF_TRUSTED_ORIGINS=https://YOURNAME.pythonanywhere.com
EOF
```

Generate a strong secret key with:

```bash
python -c "from django.core.management.utils import get_random_secret_key; print(get_random_secret_key())"
```

## 5. Build the database and content

```bash
python manage.py migrate
python manage.py createsuperuser        # your admin login

# Seed the catalog (run the seed_* commands you want -- see README.md for the
# full list). For example, the five-tier practice exams for every subject:
python manage.py seed_ged_math_exam
python manage.py seed_ged_math_exam_level2
# ...through level5, and the rla/science/social exam commands, etc.

python manage.py collectstatic --no-input
```

> Tip: the entire course catalog is generated by `seed_*` commands, so you can
> rebuild it on the server without copying the database.

## 6. Configure the Web app

1. Go to the **Web** tab → **Add a new web app** → **Manual configuration** →
   **Python 3.11**.
2. **Virtualenv:** set the path to `/home/YOURNAME/ged-sat-lms/venv`.
3. **Source code / Working directory:** `/home/YOURNAME/ged-sat-lms`.
4. **WSGI configuration file:** click it and replace the contents with:

   ```python
   import os, sys
   from dotenv import load_dotenv

   path = '/home/YOURNAME/ged-sat-lms'
   if path not in sys.path:
       sys.path.insert(0, path)

   load_dotenv(os.path.join(path, '.env'))
   os.environ['DJANGO_SETTINGS_MODULE'] = 'config.settings'

   from django.core.wsgi import get_wsgi_application
   application = get_wsgi_application()
   ```

5. **Static files mapping** (optional but faster than WhiteNoise on PythonAnywhere):
   - URL: `/static/`  →  Directory: `/home/YOURNAME/ged-sat-lms/staticfiles`
   - (WhiteNoise will also serve them if you skip this.)
6. In the Web tab, enable **Force HTTPS**.
7. Click the big green **Reload** button.

Visit `https://YOURNAME.pythonanywhere.com` — you're live.

---

## Updating the site later

```bash
cd ~/ged-sat-lms
source venv/bin/activate
git pull
pip install -r requirements.txt          # if dependencies changed
python manage.py migrate                  # if models changed
python manage.py collectstatic --no-input # if static/admin changed
# then click "Reload" in the Web tab
```

## Notes

- **Database:** SQLite lives at `db.sqlite3` on the persistent disk, so accounts
  and progress are kept across reloads. Back it up by downloading the file
  occasionally.
- **AI features:** if you use the Claude features, also add
  `ANTHROPIC_API_KEY=...` to `.env`.
- **Free tier:** the app may sleep when idle and has a daily CPU allowance; upgrade
  to a paid tier for an always-on custom domain.
- **Other hosts:** the same env vars (`DJANGO_SECRET_KEY`, `DJANGO_DEBUG`,
  `DJANGO_ALLOWED_HOSTS`, `DJANGO_CSRF_TRUSTED_ORIGINS`) and the included
  `gunicorn`/`whitenoise` setup also work on Render, Railway, or a VPS.
